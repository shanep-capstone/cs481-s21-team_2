User story Reading and Saving warehouse inventory file:

## User Story

As a user of the warehouse management program/worker in a warehouse, I want to be able to enter data through an entry file to show the current inventory of my warehouse, 
use the program and save in a file/update the inventory with the operations I have done
 

## Tasks

1. Read an entry file, acknowledge the file has been read
2. Check if the file follow the specific format required, display guidelines to the user if not
3. After the file entry has been read, select the relevant data and create a warehouse inventory database 
4. Create a function to display the warehouse inventory in a readable way for the user
5. Once the user is done using the program, create a file with the updated inventory

## Definition of Done

- Task 1 DOD - I will know that I am done when I can read a file with no errors and when a message prompt on the console telling the file was entered with success
- Task 2 DOD - I will know that I am done if after entering a file witht he good format no message prompt on the console and a message explaining the required format prompt if the file is not under the right format
- Task 3 DOD - I will know that I am done when the program select the relevant data form the file to parse them in the database
- Task 4 DOD - I will know that I am done when I can display on the console a clear representation / picture of the current inventory in the wharehouse
- Task 5 DOD - I will know that I am done when after I finished using the program I can save my data and update these change in my file
